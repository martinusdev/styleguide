§ 01

Čo je Model Context Protocol?

Otvorený štandard pre prepojenie AI s reálnym svetom.

Veľké jazykové modely vedia rozprávať, ale samé osebe sú v izolácii. Nevidia naše súbory, databázy, ani styleguide. MCP je protokol, ktorý túto bariéru ruší — definuje, ako môže LLM volať externé nástroje a získavať dáta z reálneho prostredia.

Pred MCP každý vendor (OpenAI, Google, Anthropic) riešil prepojenie modelu s nástrojmi po svojom — proprietárne function calling API. Výsledok? Roztrieštený ekosystém, integrácie sa nedali zdieľať medzi modelmi a každá appka si musela písať vlastné lepidlo.

V novembri 2024 Anthropic publikoval MCP ako open-source, vendor-neutrálny štandard. Princíp je rovnaký ako pri Language Server Protocol (LSP) pre IDE: jeden protokol, ktorý hovoria všetci, a každý nástroj sa stačí napojiť raz.

Pozn. Inšpiráciou pre MCP bol Language Server Protocol — ten istý prístup, ktorý umožnil, že rovnaký jazykový server beží vo VS Code, Vim, IntelliJ aj Sublime.

§ 02

MCP server vie ponúkať tri veci.

Tools, resources, prompts — tri primitíva, z ktorých sa skladá každý server.

MCP nedefinuje len „funkcie pre AI". Definuje tri stavebné bloky, ktoré môže server ponúknuť. Každý rieši inú potrebu — od aktívnych volaní cez pasívne čítanie dát až po pripravené šablóny.

Náš styleguide MCP server zatiaľ ponúka len tools. Ostatné dve primitíva sú v MCP špecifikácii rovnocenné, ale nie sú pre nás zatiaľ potrebné. Stojí za to ich poznať — ovplyvňujú, ako navrhneme ďalšie features.

Definícia Primitívum = entita, ktorú server vystavuje cez MCP protokol. Klient (Claude) ich objaví cez tools/list, resources/list, prompts/list.
U nás
01 · Aktívne

Tools

Funkcie, ktoré LLM volá s parametrami. Server ich vykoná a vráti výsledok. Najpoužívanejšie primitívum.

napr. get_component(name: "buttons")
02 · Pasívne

Resources

Dáta, ktoré server vystavuje na čítanie — súbory, databázové záznamy, logy. LLM ich číta podľa URI.

napr. file:///app/styles/main.scss
03 · Šablóny

Prompts

Pripravené prompt šablóny s parametrami, ktoré klient ponúkne používateľovi ako rýchle akcie.

napr. "Vytvor formulár pre {{entita}}"

A ako tieto primitíva putujú medzi klientom a serverom?

MCP definuje dva transporty. Lokálny stdio (server beží ako proces, komunikuje cez pipy) a vzdialený HTTP s server-sent events. Aplikačná vrstva nad nimi je JSON-RPC 2.0 — známy, jednoduchý.

Tip Pre lokálny vývoj — stdio. Nulová konfigurácia, žiadne porty. Pre tímové zdieľanie — HTTP server.
§ 03

Anatómia jedného volania.

Čo sa stane medzi promptom a odpoveďou.

Keď napíšeš Claudeovi „vygeneruj mi product card", odohrá sa pod kapotou krátky tanec. Stojí za to vidieť ho celý — pomôže to pochopiť, prečo niektoré veci sú rýchle a iné pomalé.

Kľúčové Discovery (kroky 1–2) sa deje raz — pri pripojení. Tool call (4–5) môže prebehnúť mnoho ráz počas jednej konverzácie.

§ 04

Čo funguje, čo nie.

Best practices, ktoré sme si overili pri stavbe vlastného servera.

MCP je jednoduchý protokol, ale postaviť dobrý server je iný príbeh. LLM nečíta kód — číta popisy toolov. To, ako tooly pomenuješ a popíšeš, určuje, či ich Claude bude vedieť rozumne použiť.

Heuristika Ak tvoj tool popis znie ako dobrá API dokumentácia, LLM ho použije správne. Ak znie ako šifra, bude tipovať.
✓

Funguje

  • 01 · Názvy Stručné, popisné, slovesné — get_component, list_icons. LLM ich číta ako API.
  • 02 · Schemas Striktné JSON schema pre parametre. Enum hodnoty, povinné polia, popisy. Claude tomu rozumie.
  • 03 · Cache Drahé fetche cachovať s rozumným TTL. My máme 5 minút — kompromis medzi čerstvosťou a rýchlosťou.
  • 04 · Discovery Samostatný tool pre získanie schemy (get_component_info) — lacnejšie ako skúšať parametre.
  • 05 · Chyby Štruktúrované error odpovede s návrhom riešenia. Náš get_icon pri chybe vráti najbližšie alternatívy.
  • 06 · Kurácia Starostlivo vybraný subset, nie všetko. Vystavujeme len ~186 ikoniek, nie celé Font Awesome Pro.
✗

Nefunguje

  • 01 · Príliš veľa Stovky toolov v jednom serveri — LLM sa stratí v možnostiach a začne náhodne tipovať.
  • 02 · Vágne názvy do_thing, process_data, handle_request — bez kontextu LLM netuší, kedy ich má použiť.
  • 03 · Mega-tool Jeden tool na všetko s 30 parametrami a režimami. Lepšie ich rozdeliť do menších, jednoúčelových.
  • 04 · Bloby Vracanie obrovských nestrukturovaných JSON-ov. Zaberie kontext, LLM si z toho ťažko niečo zoberie.
  • 05 · Žiadne príklady Popis bez ukážkového použitia. LLM síce zvládne abstraktný popis, ale s príkladom je úspešnosť výrazne vyššia.
  • 06 · Tichý fail Tool vráti prázdny string alebo null bez vysvetlenia. LLM si myslí, že všetko prebehlo OK, a používateľ dostane nezmysel.
§ 05

Prečo sme to postavili?

Bez kontextu Claude komponenty vymýšľa.

Náš styleguide nie je v Claudeových tréningových dátach. Bez MCP teda Claude pri otázke „daj mi tlačidlo" generuje presvedčivo vyzerajúce HTML — také, aké videl na tisícoch iných webov. Vyzerá to dobre, ale nie je to náš kód.

Náklady Bez MCP každý developer ručne kontroluje a opravuje vygenerovaný kód. S MCP serverom ho rovno použije.
Stav A — bez MCP

Hallucinated kód

  • Claude navrhuje komponenty zo svojej hlavy
  • Vymyslené CSS triedy, nesprávna štruktúra
  • Vývojár musí ručne opravovať každý fragment
  • Každý má svoju verziu „toho istého" tlačidla
  • Žiadny vzťah ku skutočnému styleguide
Stav B — s MCP

Kanonický kód

  • Claude sa opýta nášho servera
  • Vráti skutočný HTML zo styleguide
  • Správne triedy, správna štruktúra — hneď
  • Jeden zdroj pravdy pre AI aj ľudí
  • Update styleguide = update pre AI naraz
§ 06

Osem toolov, osem otázok.

Každý tool rieši jednu konkrétnu otázku Claudea o našom dizajn systéme.

▸get_setup

HTML tagy pre načítanie všetkých styleguide assetov — CSS, JS, fonty. Vracia správnu verziu podľa aktuálneho buildu.

language

▸get_starter_template

Kompletný HTML dokument pripravený na použitie — so všetkými assetmi a správnou štruktúrou. Quick start za jeden call.

language title

▸get_component

Renderuje HTML fragment komponentu s danými parametrami. Vráti aj zoznam závislostí (requires) — ktoré bundly treba načítať.

name ...parametre

▸list_components

Vypíše všetky dostupné komponenty s popisom a parametrami. Základ pre discovery — od neho začína každá konverzácia.

type

▸get_component_info

Detail jedného komponentu: schéma parametrov, typy, enum hodnoty, povinné / nepovinné. Lacnejšie ako probovať volaním get_component.

name

▸list_icons

Zoznam Font Awesome ikoniek zo starostlivo vybraného setu — ~186 ks v piatich štýloch. Nie celé FA Pro, len to, čo reálne používame.

style

▸get_icon

HTML snippet pre jednu ikonku. Ak ikonka neexistuje, server navrhne najbližšie alternatívy (edit distance).

name style

▸get_utilities

Referencia utility tried: flex, spacing, display, text, colors, visibility, sizing — s našimi vlastnými breakpointmi (-s/-m/-l/-xl).

category

internal onlyV lokálnom Docker mode sú navyše dostupné tooly, ktoré čítajú priamo zo zdrojov repozitára: list_scss_utilities, find_scss_class, get_data_structure. Externé pripojenie ich nemá — bezpečnosť a kontrola toho, čo vidí publikum.

§ 07

Jeden server, dva módy.

Rovnaký codebase, dve prostredia: externé fetch z produkcie cez HTTP, alebo lokálne čítanie zo súborov.
External · default

External mode

beží kdekoľvek · npm · Docker · vzdialený server

Server nemá prístup k repozitáru. Pri každom requeste si stiahne aktuálne dáta z mrtns.sk/assets a 5 minút si ich drží v cache. Vždy konzistentné s produkciou.

Claude→ MCP server→ mrtns.sk

Použiť keď: chceš zdieľať server pre celý tím, beží na cloude, alebo nemáš lokálny checkout.

Internal · dev

Internal mode

aktivuje sa nastavením MARTINUS_STYLEGUIDE_APP_DIR

Server číta priamo zo súborov repozitára. Žiadne fetche, zmena v app/mcp-components.yaml je vidieť okamžite — bez deployu, bez čakania na cache.

Claude→ MCP server→ app/*.yaml

Použiť keď: vyvíjaš nové komponenty alebo upravuješ schemu — chceš okamžitý feedback.

Ako začať.

Lokálne cez Docker, alebo vzdialený server pre celý tím.

Lokálne (Docker)

# 1. Spustí MCP server v kontajneri
./mcp.sh

# 2. Pridať do Claude Code konfigurácie
#    ~/.config/claude/claude_desktop_config.json
{
  "mcpServers": {
    "martinus-styleguide": {
      "command": "docker",
      "args": [
        "exec", "-i",
        "martinus-styleguide-mcp",
        "node",
        "/mcp-server/src/index.js"
      ]
    }
  }
}
