§ 01 · Problém

Claude nepozná naše komponenty.

Náš styleguide nikdy nevidel. Bez kontextu generuje presvedčivo vyzerajúci HTML — taký, aký videl na tisícoch iných webov.

Bez MCP

  • −
    Vymyslené CSS triedy — názvy, ktoré v styleguide neexistujú.
  • −
    Nesprávna štruktúra — wrappers a hierarchia tagov ad-hoc.
  • −
    Ručná oprava každého fragmentu pred použitím.
  • −
    Žiadny vzťah k reálnemu styleguide.

S MCP

  • +
    Skutočný HTML zo styleguide, ready to paste.
  • +
    Správne triedy, správna štruktúra — hneď.
  • +
    Jeden zdroj pravdy pre AI aj ľudí.
  • +
    Update styleguide = update pre AI naraz.

§ 03 · Primitíva

MCP server ponúka tri veci.

Tools, resources, prompts. My zatiaľ používame len prvé.

U nás 01 · Aktívne

Tools

Funkcie, ktoré LLM volá s parametrami. Server ich vykoná a vráti výsledok.

get_component("buttons")
Pasívne 02 · Dáta

Resources

Dáta, ktoré server vystavuje na čítanie — súbory, logy. LLM ich číta podľa URI.

file:///app/main.scss
Šablóny 03 · Prompty

Prompts

Pripravené prompt šablóny s parametrami — klient ich ponúka ako rýchle akcie.

"Vytvor formulár pre {{x}}"

§ 05 · Best practices

Čo funguje, čo nie.

LLM nečíta kód — číta popisy toolov. Pomenovanie a popis určuje úspešnosť.

Funguje

  • +
    Stručné popisné názvy — get_component.
  • +
    Striktné JSON schema s enum hodnotami.
  • +
    Cache drahých fetchov s TTL.
  • +
    Štruktúrované chyby s návrhom riešenia.
  • +
    Discovery tool pre získanie schemy.

Nefunguje

  • −
    Stovky toolov — LLM sa stratí.
  • −
    Vágne názvy ako do_thing.
  • −
    Jeden mega-tool s 30 parametrami.
  • −
    Obrovské blob odpovede bez štruktúry.
  • −
    Tichý fail — návrat null bez vysvetlenia.
§ 06 · Kedy použiť

Kedy MCP, kedy nie.

Nie každý kontextový problém potrebuje vlastný server. Tu sú konkrétne príklady.

Vhodné

  • +
    Interný design system — komponenty, tokens, ikonky (čo robíme my).
  • +
    Interné dáta a API — customer lookup, ticket history, štruktúra firmy.
  • +
    External SaaS s API kľúčom — Sentry, Linear, Slack, Grafana.
  • +
    Codebase navigation — symbol search, file deps, recent commits.

Nevhodné

  • −
    Verejné API LLM pozná — Stripe, AWS, npm, MDN.
  • −
    Čistá výpočtová logika — regex, parsing, formátovanie.
  • −
    Heavy data dumps — celé DB tabuľky / log súbory.
  • −
    Real-time produkčné mutácie — bez human-in-the-loop.
§ 07 · Bezpečnosť

Štyri vrstvy obrany.

MCP server nie je len helper — môže byť útočným vektorom. Treba myslieť od transportu po dáta.

01 · Transport

Šifrované spojenie

stdio je process-isolated (lokálny Docker = bezpečný default). HTTP mode potrebuje TLS — Let's Encrypt + reverse proxy.

02 · Auth

Autentifikácia klientov

stdio: process boundary stačí — kto má docker exec, má prístup. HTTP: API kľúč v hlavičke alebo OAuth scope.

03 · Tool design

Validácia inputov

Striktné JSON schema s enum hodnotami. Žiadny path traversal, žiadny SQL injection cez user-supplied strings.

04 · Operations

Audit + human-in-the-loop

Logy čo / kedy / kým bolo zavolané. Mutácie produkcie len s explicitnou konfirmáciou. Rate limit pre runaway agents.

§ 09 · Architektúra

Jeden server, dva módy.

Rovnaký codebase — len iný zdroj dát.

External mode default · beží kdekoľvek

Server fetchuje aktuálne dáta z mrtns.sk/assets, drží ich v cache 5 minút. Vždy konzistentné s produkciou.

Claude → MCP server → mrtns.sk
Internal mode dev · číta z repozitára

Aktivuje sa env premennou. Server číta YAML súbory priamo — zmeny vidno okamžite bez deploya.

Claude → MCP server → app/*.yaml
§ 10 · Build pipeline

Ako sa dáta
dostanú na server.

YAML je zdroj pravdy. JSON je len distribučný formát.

YAML
app/mcp-*.yaml
→
postbuild
emit-artifacts
→
JSON
dist/mcp-*.json
→
CDN
mrtns.sk/assets
# app/mcp-components.yaml
buttons:
  template: |-
    <button class="btn">{{label}}</button>
  params:
    label:
      type: string
      required: true
    variant:
      type: enum
      values: [primary, secondary, ghost]
→ 
// dist/mcp-components.json
{
  "buttons": {
    "template": "<button class=\"btn\">...",
    "params": {
      "label": {
        "type": "string",
        "required": true
      },
      "variant": {
        "type": "enum",
        "values": ["primary","secondary","ghost"]
      }
    }
  }
}

Zdrojom pravdy je YAML (čitateľný, s komentármi, multiline templates). Po ./build.sh sa emituje JSON, ktorý MCP server fetchuje z CDN a cachuje 5 minút.

        "exec", "-i",

  }
