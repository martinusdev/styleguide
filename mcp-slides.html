§ 03 Primitíva protokolu

MCP server ponúka tri veci.

Tools, resources, prompts. My zatiaľ používame len prvé.

U nás
01 · Aktívne

Tools

Funkcie, ktoré LLM volá s parametrami. Server ich vykoná a vráti výsledok.

napr. get_component(name: "buttons")
02 · Pasívne

Resources

Dáta, ktoré server vystavuje na čítanie — súbory, log záznamy. LLM ich číta podľa URI.

napr. file:///app/styles/main.scss
03 · Šablóny

Prompts

Pripravené prompt šablóny s parametrami, ktoré klient ponúka ako rýchle akcie.

napr. "Vytvor formulár pre {{entita}}"
§ 04 Lifecycle tool callu

Anatómia jedného volania.

Discovery sa deje raz pri pripojení. Tool call môže prebehnúť mnohokrát počas konverzácie.

initialize

Handshake medzi klientom a serverom. Výmena verzií, capabilities.

tools/list

Klient si vypýta zoznam dostupných toolov so schemami parametrov.

LLM rozhoduje

Claude na základe promptu vyberá, ktorý tool zavolá a s akými argumentmi.

tools/call

Klient pošle request s názvom toolu a parametrami. Server validuje a vykoná.

response

Server vráti štruktúrovaný výsledok, ktorý LLM použije v odpovedi.

§ 05 Best practices

Čo funguje, čo nie.

LLM nečíta kód — číta popisy toolov. To, ako ich pomenuješ a popíšeš, určuje úspešnosť.

+

Funguje

  • Stručné, popisné názvy — get_component, list_icons.
  • Striktné JSON schema s enum hodnotami a popismi.
  • Cache drahých fetchov s rozumným TTL (5 min).
  • Štruktúrované chyby s návrhom riešenia.
  • Discovery tool oddelený od execution toolu.
−

Nefunguje

  • Stovky toolov v jednom serveri — LLM sa stratí.
  • Vágne názvy ako do_thing, process_data.
  • Jeden mega-tool s 30 parametrami a režimami.
  • Obrovské nestrukturované blob odpovede.
  • Tichý fail — návrat null bez vysvetlenia.
§ 06 Naša implementácia

Osem toolov, osem otázok.

Každý tool rieši jednu konkrétnu otázku Claudea o našom dizajn systéme.

get_setup

HTML tagy pre načítanie všetkých styleguide assetov.

get_starter_template

Kompletný HTML dokument pripravený na použitie.

get_component

Renderuje HTML fragment komponentu s parametrami.

list_components

Vypíše všetky dostupné komponenty s popisom.

get_component_info

Detail jedného komponentu — schéma parametrov.

list_icons

~186 Font Awesome ikoniek v 5 štýloch.

get_icon

HTML snippet pre ikonku, s návrhom alternatív.

get_utilities

Utility triedy: flex, spacing, colors, sizing…

§ 07 Architektúra

Jeden server, dva módy.

Rovnaký codebase — len iný zdroj dát.

External · default

External mode

beží kdekoľvek · fetch z produkcie

Server nemá prístup k repozitáru. Fetchuje aktuálne dáta z mrtns.sk/assets, drží ich v cache 5 minút. Vždy konzistentné s produkciou.

Claude→ MCP server→ mrtns.sk
Internal · dev

Internal mode

číta priamo z repozitára

Aktivuje sa env premennou MARTINUS_STYLEGUIDE_APP_DIR. Číta YAML súbory priamo — zmeny vidno okamžite bez deployu.

Claude→ MCP server→ app/*.yaml
